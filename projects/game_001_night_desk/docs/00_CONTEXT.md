# Night Desk 项目上下文

## 项目定位

- 项目编号：`game_001_night_desk`
- 中文名：`旅馆前台：请勿让他们入住`
- 英文名：`Night Desk`
- 平台：HTML5、itch.io、`Play in browser`
- 类型：规则判断、恐怖打工模拟、多结局互动小游戏
- 目标时长：10-15 分钟
- 技术栈：HTML、CSS、Vanilla JavaScript
- 当前阶段：在现有最小可玩骨架上补齐正式 MVP 内容

玩家扮演小旅馆夜班前台，根据逐渐互相矛盾的规则判断客人是否可以入住。老板会施加经营压力，客人会越来越异常。玩家需要在赚钱、避免投诉、控制污染、维持理智和保持老板信任之间做判断。

## 核心循环

```text
客人出现
-> 读取今日规则
-> 查看客人特征与登记信息
-> 选择允许入住 / 拒绝入住 / 报警 / 假装系统故障
-> 更新金钱、投诉、污染值、理智、老板信任
-> 进入下一位客人或下一晚
-> 触发结局
```

## 核心数值

- `money`：收入，太低会被老板开除。
- `complaints`：投诉值，拒绝正常客人会增加。
- `corruption`：污染值，放进异常客人会增加。
- `sanity`：理智值，看到异常或做错判断会降低。
- `bossTrust`：老板信任，影响经营类结局。

## MVP 边界

### 必须实现

- 7 个夜晚。
- 约 15 个客人。
- 每晚 2-4 条规则。
- 固定前台界面。
- 极简暗色 UI。
- 四个玩家操作按钮：
  - `Allow Check-in`
  - `Refuse`
  - `Call Police`
  - `System Error`
- 至少 5 个结局：
  - 被开除
  - 旅馆吃掉你
  - 老板结局
  - 逃离结局
  - 真结局

### 绝对不做

- 不做 3D。
- 不做自由移动。
- 不做背包系统。
- 不做联网。
- 不做复杂存档。
- 不做大型剧情分支树。
- 不做复杂角色立绘系统。
- 不使用大型游戏引擎。
- 不引入复杂构建工具。

## 当前阶段目标

现有代码已经具备最小可玩骨架，但当前内容仍是流程验证版本。下一阶段应在保留轻量结构的前提下，将双按钮、少量测试客人的骨架升级为正式 MVP：

1. 扩展 7 晚规则与约 15 位客人的数据结构。
2. 加入 `money`、`complaints`、`corruption`、`sanity`、`bossTrust`。
3. 将操作扩展为四个按钮，并为不同操作提供明确反馈。
4. 实现按夜晚推进和至少 5 个结局。
5. 完成桌面端与常见浏览器窗口尺寸下的手动测试。

## 后续开发阶段

1. **阶段 1：数据与状态模型**
   明确夜晚、规则、客人、操作结果、核心数值和结局条件的数据结构。
2. **阶段 2：正式 MVP 交互**
   接入四个操作、夜晚推进、状态更新和多结局。
3. **阶段 3：UI 打磨**
   保持固定前台界面，增强暗色恐怖氛围、可读性和屏幕适配。
4. **阶段 4：QA 与内容校准**
   检查边界状态、结局可达性、规则表达和 10-15 分钟体验时长。
5. **阶段 5：itch.io 发布准备**
   整理发布 zip、页面文案、标签和展示物料。
