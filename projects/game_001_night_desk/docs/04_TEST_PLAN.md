# 04 Test Plan

## 基础流程

- 页面首次加载后展示第一夜规则和五项初始数值。
- 点击“开始夜班”后显示第一位旅客的主场景、人物画像、对话框、左侧登记资料和器官检查项。
- 每位旅客只能记录一次决定。
- 四个处置按钮均能触发反馈和数值变化。
- 处理第 15 位旅客后显示结局。
- 点击“重新值班”后分数、数值和旅客索引重置。

## 极端测试用例

- 快速连续点击任意处置按钮，确认不会重复计分。
- 快速交替点击多个决定按钮，确认只接受第一次选择。
- 完整跑通真结局、被开除、旅馆吃掉你、老板结局、逃离结局。
- 刷新页面，确认从未开始状态重新载入。
- 使用窄屏尺寸，确认按钮纵向排列且文本不会溢出。
- 使用窄屏尺寸，确认左侧面板、客人场景、对话框和检查面板改为可读单列。
- 临时清空某类数据，确认需要补充数据保护逻辑的范围。

## 视觉与素材测试

- `assets/concepts/` 中 concept 图仅作为参考，不被直接写死为动态 UI。
- 没有独立 portrait 时，客人使用 CSS 生成剪影；当前 15 位 MVP 客人应全部有 portrait。
- 有 `inspection` 数据时，画面上显示器官异常标记，检查面板显示异常程度。
- 正常客人也要显示检查项，避免玩家只根据“有标记”机械判断。

## 音频与图标测试

- 页面加载后确认 `favicon.ico` 返回 200，不再出现默认 favicon 404。
- 点击“开始夜班”后确认雨声循环开始。
- 来客出现时确认门铃音触发。
- 点击任意处置按钮时确认按钮音触发。
- 危险或错误异常处置时确认低频异常音触发。

## 控制台排查

- `NIGHT_DESK_NIGHTS is undefined`：检查 `data/rules.js` 是否先于 `main.js` 加载。
- 数据字段缺失：检查旅客对象是否包含 `details`、`inspection`、`visual`、`speechClues`、`correctDecision` 和四种 `outcomeText`。
- 结局错误：检查 `endings.js` 的 `conditions` 是否能匹配当前数值。
- 音频无法播放：确认用户已点击“开始夜班”，浏览器通常会阻止未交互前的自动播放。
