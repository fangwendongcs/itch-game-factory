# 04 Test Plan

## 基础流程

- 页面首次加载后展示三条规则。
- 点击“开始夜班”后显示第一位旅客。
- 每位旅客只能记录一次决定。
- 处理第三位旅客后显示结局。
- 点击“重新值班”后分数和旅客索引重置。

## 极端测试用例

- 快速连续点击“允许入住”或“拒绝入住”，确认不会重复计分。
- 快速交替点击两个决定按钮，确认只接受第一次选择。
- 完整跑通 `3/3`、`2/3` 和 `0-1/3` 三种结局。
- 刷新页面，确认从未开始状态重新载入。
- 使用窄屏尺寸，确认按钮纵向排列且文本不会溢出。
- 临时清空某类数据，确认需要补充数据保护逻辑的范围。

## 控制台排查

- `NIGHT_DESK_RULES is undefined`：检查 `data/rules.js` 是否先于 `main.js` 加载。
- 数据字段缺失：检查旅客对象是否包含 `details`、`correctDecision` 和反馈文案。
- 结局错误：检查 `endings.js` 是否按 `minScore` 从高到低排列。
