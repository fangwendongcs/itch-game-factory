# 02 PRD MVP

## MVP 功能清单

- 开始夜班。
- 展示 7 个夜晚对应的入住规则。
- 逐个展示旅客资料、观察特征和话语线索。
- 玩家选择四种处置：
  - `Allow Check-in`
  - `Refuse`
  - `Call Police`
  - `System Error`
- 根据数据中的正确决定显示即时反馈。
- 记录正确判断数和五项核心数值。
- 全部旅客处理后，根据正确数与数值状态显示多结局。
- 支持重新开始。

## 数值与状态定义

| 状态 | 说明 |
| --- | --- |
| `currentIndex` | 当前旅客索引 |
| `correctCount` | 正确判断数量 |
| `locked` | 防止同一旅客被重复点击 |
| `money` | 收入，过低会进入被开除风险 |
| `complaints` | 投诉值，错误拒绝正常客人会增加 |
| `corruption` | 污染值，错误放行异常客人会增加 |
| `sanity` | 理智值，异常事件和错误判断会降低 |
| `bossTrust` | 老板信任，影响经营类结局 |
| `correctDecision` | 每位旅客的正确决定：`allow`、`refuse`、`police` 或 `error` |
| `conditions` | 结局触发条件，基于正确数和核心数值判断 |

## 音效 / UI 清单

- 当前版本：纯 CSS 暗色前台 UI，无素材依赖。
- 后续最低素材：大堂环境底噪、按钮盖章音、门铃、异常低频音。
- 后续最低图片：旅客照片或剪影、旅馆徽标、纸张纹理。

## 绝对不做清单

- 不做账号、云存档或排行榜。
- 不做自由走动、战斗或背包。
- 不做联网请求。
- 不做大量动态动画。
- 不做复杂经营系统，当前只保留轻量夜晚推进和数值反馈。

## MVP 验收标准

- 浏览器打开后无控制台报错。
- 15 位旅客可以完整处理。
- 快速重复点击不会重复计分。
- 至少 5 个结局可以通过不同数值状态匹配。
- 手机窄屏下按钮仍可点击，规则和资料可阅读。
