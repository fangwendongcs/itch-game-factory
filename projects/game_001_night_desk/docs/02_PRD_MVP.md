# 02 PRD MVP

## MVP 功能清单

- 开始夜班。
- 展示本轮固定规则。
- 逐个展示旅客资料和一句请求。
- 玩家选择“允许入住”或“拒绝入住”。
- 根据数据中的正确决定显示即时反馈。
- 记录正确判断数。
- 全部旅客处理后，根据正确数显示结局。
- 支持重新开始。

## 数值与状态定义

| 状态 | 说明 |
| --- | --- |
| `currentIndex` | 当前旅客索引 |
| `correctCount` | 正确判断数量 |
| `locked` | 防止同一旅客被重复点击 |
| `correctDecision` | 每位旅客的正确决定：`approve` 或 `reject` |
| `minScore` | 结局所需最低正确数，按从高到低配置 |

## 音效 / UI 清单

- 当前骨架：纯 CSS UI，无素材依赖。
- 后续最低素材：大堂环境底噪、按钮盖章音、门铃、异常低频音。
- 后续最低图片：旅客照片或剪影、旅馆徽标、纸张纹理。

## 绝对不做清单

- 不做账号、云存档或排行榜。
- 不做自由走动、战斗或背包。
- 不做联网请求。
- 不做大量动态动画。
- 不在 MVP 中加入多日经营系统。

## MVP 验收标准

- 浏览器打开后无控制台报错。
- 三位测试旅客可以完整处理。
- 快速重复点击不会重复计分。
- 三种得分区间都能匹配到结局。
- 手机窄屏下按钮仍可点击，规则和资料可阅读。
