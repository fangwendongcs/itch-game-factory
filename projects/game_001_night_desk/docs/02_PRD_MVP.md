# 02 PRD MVP

## MVP 功能清单

- 开始夜班。
- 展示 7 个夜晚对应的入住规则。
- 逐个展示旅客监控场景、人物画像、登记资料、器官异常检查和话语线索。
- 玩家选择四种处置：
  - `Allow Check-in`
  - `Refuse`
  - `Call Police`
  - `System Error`
- 根据数据中的正确决定显示即时反馈。
- 记录正确判断数和五项核心数值。
- 全部旅客处理后，根据正确数与数值状态显示多结局。
- 支持重新开始。

## 数值与状态定义

| 状态 | 说明 |
| --- | --- |
| `currentIndex` | 当前旅客索引 |
| `correctCount` | 正确判断数量 |
| `locked` | 防止同一旅客被重复点击 |
| `money` | 收入，过低会进入被开除风险 |
| `complaints` | 投诉值，错误拒绝正常客人会增加 |
| `corruption` | 污染值，错误放行异常客人会增加 |
| `sanity` | 理智值，异常事件和错误判断会降低 |
| `bossTrust` | 老板信任，影响经营类结局 |
| `correctDecision` | 每位旅客的正确决定：`allow`、`refuse`、`police` 或 `error` |
| `conditions` | 结局触发条件，基于正确数和核心数值判断 |
| `visual` | 客人的场景、剪影、威胁等级或未来 portrait 路径 |
| `inspection` | 客人的器官/异常部位检查项，包含异常程度 |

## 音效 / UI 清单

- 当前版本：冷绿灰监控风格 UI，左侧放规则、数值、来客档案和操作按钮，主区域突出旅馆场景、人物画像和对话框。
- 当前视觉资产：zip 中 concept 图已放入 `assets/concepts/`，仅作为风格参考；15 张客人 portrait 已放入 `assets/portraits/` 并映射到来客数据。
- 当前音效：雨声循环、按钮音、门铃、异常低频音。
- 后续音效优化：替换为更精致的正式素材，增加音量设置或静音开关。
- 后续最低图片：正式前台/走廊背景、旅馆徽标、纸张纹理，以及替换带烘焙棋盘格背景的透明人物图。

## 绝对不做清单

- 不做账号、云存档或排行榜。
- 不做自由走动、战斗或背包。
- 不做联网请求。
- 不做大量动态动画。
- 不做复杂经营系统，当前只保留轻量夜晚推进和数值反馈。

## MVP 验收标准

- 浏览器打开后无控制台报错。
- favicon 正常加载，不出现默认 `/favicon.ico` 404。
- 点击“开始夜班”后音频可解锁，雨声、按钮、门铃和异常音可按事件触发。
- 点击“开始夜班”后能看到主场景中的客人形象、对话框，以及左侧来客档案中的器官检查项。
- 15 位旅客可以完整处理。
- 快速重复点击不会重复计分。
- 至少 5 个结局可以通过不同数值状态匹配。
- 手机窄屏下按钮仍可点击，规则和资料可阅读。
